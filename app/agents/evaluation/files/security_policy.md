# Security Policy – Best Practices

## 1. Purpose
This policy establishes the minimum security requirements and best practices to protect company information, systems, and assets from unauthorized access, disclosure, alteration, or destruction.

## 2. Scope
This policy applies to all employees, contractors, consultants, temporary staff, and third‑party partners who access company systems, networks, or data.

---

## 3. Security Best Practices

### A. Access Control
- Use **unique user accounts**; sharing credentials is prohibited.
- Apply **Multi‑Factor Authentication (MFA)** for all critical systems.
- Access must follow **least privilege** and **role‑based access control (RBAC)**.
- Review user permissions **quarterly** and promptly remove unused access.

### B. Password Management
- Passwords must be **strong**, using a mix of characters and a minimum length of 12+.
- Change passwords only if compromise is suspected.
- Store passwords in **approved password managers**; never write them down or email them.

### C. Device & Endpoint Security
- Enable **automatic updates** for operating systems and applications.
- Use **company‑approved antivirus/EDR** tools.
- Lock screens after **5 minutes of inactivity**.
- Encrypt laptops, mobile devices, and removable media.

### D. Network Security
- Connect to corporate systems only via secure networks; use **VPN** when remote.
- Public Wi‑Fi requires VPN usage.
- Unauthorized network devices are prohibited.

### E. Data Protection
- Classify data according to sensitivity (e.g., Public, Internal, Confidential).
- Follow **data minimization**: collect and store only what is necessary.
- Encrypt confidential data **in transit** and **at rest**.
- Use approved platforms for file sharing; avoid personal cloud accounts.

### F. Email & Communication Security
- Treat unsolicited emails with caution; avoid unknown links or attachments.
- Verify sender identity before sharing sensitive information.
- Report suspected phishing attempts immediately.

### G. Physical Security
- Display company ID badges in secure areas.
- Lock workstations and secure documents when unattended.
- Report lost or stolen devices within **1 hour**.

### H. Incident Reporting
- Immediately report any suspected security incidents, data breaches, malware infections, unauthorized access, or unusual system behavior.
- Do **not** attempt to fix major incidents yourself; follow escalation procedures.

---

## 4. Compliance
Non‑compliance with this policy may result in disciplinary action, restricted access, or termination. Compliance is monitored through audits, system checks, and periodic reviews.
